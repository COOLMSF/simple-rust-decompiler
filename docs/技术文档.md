# simple-decompiler 技术文档

## 1. 什么是反编译器

编译器把 C 变成二进制，反编译器反过来：二进制 → C 代码。

## 2. 整体流程

```
二进制文件 → 加载 → 反汇编 → 转IR → 分析 → 生成C代码
```

6 个源码文件对应 6 个步骤：

| 文件 | 功能 |
|------|------|
| `binary/mod.rs` | 解析 ELF/PE |
| `disasm/mod.rs` | 机器码→汇编 |
| `ir/mod.rs` | IR 数据结构 |
| `ir/lift.rs` | 汇编→IR |
| `analysis/mod.rs` | 控制流分析 |
| `codegen/mod.rs` | IR→C代码 |

## 3. 加载二进制 (`binary/mod.rs`)

用 goblin 库解析 ELF/PE，提取代码段、符号表、入口点。

核心结构：
- `LoadedBinary`：架构、入口点、段列表、符号表
- `Section`：段名、虚拟地址、原始数据、是否可执行

## 4. 反汇编 (`disasm/mod.rs`)

用 iced-x86 把机器码翻译成汇编指令。

函数发现：符号表 + 扫描 CALL 指令。
函数体解码：递归下降，遇到 ret/jmp 停止，条件跳转追踪两条路径。

## 5. 中间表示 IR (`ir/mod.rs`)

汇编太复杂，先转成简化的 IR：

- **VReg**：虚拟寄存器（v0, v1, v2...），数量无限
- **Expr**：纯表达式，如 `v1 + 42`、`*(addr)`
- **Stmt**：有副作用的语句，如 `v1 = expr`、`*addr = val`、`call foo()`
- **BasicBlock**：基本块，直线代码，末尾有跳转

## 6. 指令转换 (`ir/lift.rs`)

核心是 Lifter 结构，维护 `reg_map`（真实寄存器→虚拟寄存器映射）。

处理流程：
1. 读寄存器 → 查 reg_map，没有就创建新 VReg（参数）
2. 写寄存器 → 创建新 VReg，更新 reg_map
3. 内存操作 → Load/Store 语句
4. 运算 → BinOp 表达式
5. 跳转 → Jump/Branch 语句

支持的指令：mov, add, sub, imul, idiv, and, or, xor, shl, shr, cmp, test, jmp, je, jne, call, ret, push, pop, lea 等。

## 7. 程序分析 (`analysis/mod.rs`)

三个分析：

**CFG（控制流图）**：基本块为节点，跳转为边。
**支配树**：Cooper 算法，块 A 支配块 B = 到 B 必经 A。
**循环检测**：回边（目标支配源）→ 反向 BFS 收集循环体。

## 8. 生成 C 代码 (`codegen/mod.rs`)

按支配树递归生成结构化代码：

- 2 个后继 → if/else
- 回边 → while 循环
- 1 个后继 → 顺序执行
- 复杂情况 → goto

变量声明按需生成，类型用 uint32_t/uint64_t。

## 9. 完整示例

输入 C 代码：
```c
int add(int a, int b) { return a + b; }
```

编译后反编译输出：
```c
uint64_t add(uint64_t param_rdi, uint64_t param_rsi) {
    uint32_t v5;
    v5 = ((uint32_t)param_rsi);
    uint32_t v6;
    v6 = ((uint32_t)param_rdi);
    uint32_t v7;
    v7 = (v5 + v6);
    uint64_t v8;
    v8 = v7;
    return v8;
}
```

## 10. 使用方式

```bash
cargo build --release
./target/release/decomp -l a.out        # 列出函数
./target/release/decomp -f add a.out    # 反编译指定函数
./target/release/decomp -o out.c a.out  # 输出到文件
```
